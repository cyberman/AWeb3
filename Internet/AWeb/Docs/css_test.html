CSS1/CSS2 Compatibility Test

This page tests AWeb's support for CSS1 and CSS2 properties via inline styles (STYLE attribute). Each section demonstrates supported CSS properties with explanatory text describing what you should see. Note that some properties have limitations due to AWeb's layout engine architecture - see the Summary section for details.

CSS Support Summary

Fully Supported CSS1/CSS2 Properties

  • text-align - left, center, right, justify
  • font-family - comma-separated font lists with generic families (serif, sans-serif, monospace)
  • font-size - absolute sizes (xx-small to xx-large) and relative sizes (smaller, larger)
  • font-style - italic, normal
  • font-weight - bold, normal, and numeric values (100-900)
  • color - hex color values (#RRGGBB format)
  • text-decoration - none, line-through (strikethrough)
  • text-transform - uppercase, lowercase, capitalize, none
  • white-space - nowrap, normal
  • line-height - numeric values (e.g., 1.0, 1.5, 2.0) for line spacing (parsed but not yet applied; see limitations below)
  • padding - shorthand property supporting 1-4 values (e.g., padding: 10px, padding: 10px 20px, padding: 10px 20px 30px, padding: 10px 20px 30px 40px); applied to left and top margins (right/bottom not supported; see limitations below)
  • background-color - hex color values (#RRGGBB format), applied as text highlight for paragraphs
  • background-image - url() format for background images
  • border - border width parsing (full rendering may be limited)
  • border-color - hex color values for table borders
  • border-style - parsed (solid, dashed, dotted, etc.)
  • list-style-type - disc, circle, square, none (for UL), decimal, lower-alpha, upper-alpha, lower-roman, upper-roman, none (for OL)
  • display: inline - for horizontal list layouts (removes bullets and prevents line breaks between items)
  • position - absolute, relative, static, fixed (parsed, rendering limited - see limitations below)
  • top, left - pixel and percentage values (parsed, rendering limited - see limitations below)
  • margin - shorthand property supporting 1-4 values (e.g., margin: 10px, margin: 10px 20px, margin: 10px 20px 30px, margin: 10px 20px 30px 40px)
  • margin-left, margin-top - pixel, percentage, and em values (fully supported)
  • margin-right, margin-bottom - pixel, percentage, and em values (parsed but rendering limited - see limitations below)
  • width, height - pixel and percentage values (for table cells and images); fully supported
  • vertical-align - top, middle, bottom, baseline (for table cells); fully supported
  • display - inline, block, none, grid
  • grid-gap - pixel values for row and column gaps (e.g., grid-gap: 4px 16px)
  • grid-template-columns - column template definitions (e.g., max-content)
  • grid-column-start, grid-column-end - grid column positioning (numeric values, e.g., grid-column-start: 2)
  • grid-row-start, grid-row-end - grid row positioning (parsed, rendering may be limited)

Partially Supported / Limitations

  • padding - Shorthand property (1-4 values) is parsed correctly, but only left and top margins are applied; right and bottom padding are not supported by AWeb's layout engine
  • position, top, left - Parsed but rendering is limited, especially within table contexts
  • margin-right, margin-bottom - Parsed but not fully applied; AWeb's layout engine has limited support for right and bottom margins on generic body elements
  • border - Width, style, and color are parsed but full border rendering may be limited
  • border-radius - Parsed but not applied (CSS3 feature)
  • float - Not yet implemented (would require element-level changes to AWeb's layout engine)
  • grid-template-columns - Parsed but full grid layout rendering is simplified (max-content is supported for column sizing)
  • grid-column-end, grid-row-start, grid-row-end - Parsed but rendering may be limited

Implementation Notes

  • All CSS properties are supported via inline styles (STYLE attribute) only
  • External stylesheets via <LINK rel="stylesheet"> are tested separately in stylesheet_test.html
  • CSS properties can be combined in a single STYLE attribute, separated by semicolons
  • Table cell styles (width, height, vertical-align, background-color, border-color) are applied to both the cell body and table cell structure, ensuring CSS overrides HTML attributes
  • Link styles (text-decoration: none, color) are properly applied to anchor elements, with text-decoration: none removing the underline pattern
  • Font properties (font-family, font-size, font-style, font-weight) work on body elements, paragraphs, divs, and table cells
  • Text properties (text-decoration, text-transform, white-space) are applied to body elements and affect text rendering
  • Background-color for paragraphs is implemented as a text highlight, allowing each paragraph to have its own distinct background color even when sharing the same body element within table cells
  • Background-image supports url() format with optional quotes and is applied to body elements
  • List-style-type is applied to UL and OL elements via inline styles, supporting disc, circle, square, none for UL and decimal, lower-alpha, upper-alpha, lower-roman, upper-roman, none for OL
  • display: inline for lists creates horizontal layouts by removing bullets and preventing line breaks between list items
  • margin-left and margin-right support "em" units for relative spacing (e.g., margin-left: 1em)
  • line-height supports numeric values (e.g., 1.5) for controlling line spacing (parsed but not yet applied; requires layout engine support)
  • padding shorthand supports 1-4 values following CSS standard (e.g., padding: 10px, padding: 10px 20px, padding: 10px 20px 30px, padding: 10px 20px 30px 40px); only left and top margins are applied
  • margin shorthand supports 1-4 values following CSS standard (e.g., margin: 10px, margin: 10px 20px, margin: 10px 20px 30px, margin: 10px 20px 30px 40px); only top and left margins are applied
  • width and height CSS properties for table cells support both pixel and percentage values (e.g., width: 100px, width: 50%)
  • vertical-align CSS property for table cells supports top, middle, bottom, and baseline values
  • Grid layout properties (display: grid, grid-gap, grid-template-columns, grid-column-start) enable two-column layouts for definition lists (DL), with DT elements in the first column and DD elements with grid-column-start: 2 in the second column
  • grid-gap supports two values (row gap and column gap) in pixel units (e.g., grid-gap: 4px 16px)
  • grid-column-start with value >= 2 applies left margin to position elements in the second column, using the column gap from the parent grid
  • Color properties (color, background-color, border-color) support hex color codes in #RRGGBB format (3-digit shorthand like #f00 is not yet supported)
  • Text-transform is applied during text parsing, transforming text before it's added to the text buffer (uppercase, lowercase, capitalize)
  • Memory management uses AWeb's pool-based allocation system (ALLOCSTRUCT/ALLOCTYPE with FREE) for proper cleanup

Not Yet Supported

  • CSS rules in <STYLE> tags (parsing and application exist for many properties, including font-family, color, text-decoration, display, grid properties, and pseudo-classes like :link, :visited, :hover)
  • CSS selectors (class, ID, element selectors) - parsing exists but application is limited
  • CSS cascade and specificity
  • CSS pseudo-classes (:hover, :visited, etc.)
  • CSS inheritance
  • Additional CSS1/CSS2 properties not listed above
  • CSS3 features (except border-radius parsing)
